# План тестовой реализации и открытые вопросы

Как буду строить тестовую среду у себя и какие ответы от вас ещё нужны, чтобы довести до полной логики.

---

## 1. Как буду реализовывать тест у себя

### 1.1 Среда и стек (уже есть)

- **Backend:** FastAPI, SQLite (при росте — PostgreSQL), папка `backend/`.
- **Frontend:** React + Vite, папка `frontend/`.
- **Запуск:** `backend` — uvicorn; `frontend` — npm run dev. Данные из папки «база для данных» и при необходимости моки в коде/БД.

### 1.2 Что делаю по шагам (порядок реализации)

| Этап | Что делаю | Мок / реальные данные |
|------|------------|------------------------|
| **1. Справочники и БД** | Таблицы: города, номенклатура (из Номенклатура.xlsx), контрагенты (из Контрагент данные.xlsx), адреса с координатами (из Адреса (2).xlsx). Скрипт загрузки из «база для данных» в БД. | Реальные файлы из папки; при отсутствии столбцов — моковые поля. |
| **2. Курсы и правки ПНЛ** | Справочник курсов по месяцам (ручная корректировка). Таблица версий правок (кто, когда, что изменил) для слоя корректировок. | Хранение в БД; экран ввода курса и просмотра правок. |
| **3. ПНЛ (тест)** | API: факт по месяцам (статьи, суммы). Источник — либо загрузка Excel (выгрузка 1010/1030), либо мок. План — загрузка одного файла БДР (вставка в «январь») и разворот плана на год. Без формул «объединение данных» — упрощённая схема: 1010+1030 → одна «разница» по статьям из справочника. | Мок статей и сумм; реальная логика после выгрузки формул «объединение данных» и свода. |
| **4. Касса и банк** | Листы/загрузка: «Касса» (1010 по дням с 1 по текущее), «Банк» (1030 по дням). API приёма загруженного Excel или ручного ввода по дням; отображение на дашборде. | Формат столбцов — по вашему отчёту 1С; при отсутствии — мок (дата, сумма, организация). |
| **5. Мотивация (новая модель)** | Таблицы: реал за 8 мес (контрагент/город, номенклатура, период, сумма с возвратами/без); планы по городам и сотрудникам; остатки по позициям; прайс. Логика: средняя реализация за 8 мес → план задаётся в тг или % к средней; один коэффициент распределения по городам/сотрудникам; проверка план ≤ остаток и при необходимости по прайсу. Экран: ввод плана, выбор «только эти города / эта номенклатура», расчёт коэф и предупреждения. | Мок реал 8 мес (~230 млн) и остатков; реальные формулы после уточнения столбцов G/J/Q/R/X и связи с «Продажи дистр». |
| **6. Дзад (дебиторка/возвраты)** | Таблица факт/план по городам из дзад; коэффициент выравнивания к плану. API и экран рядом с мотивацией. | Мок дзад; реальные данные после загрузки листа Дзад. |
| **7. Результаты продаж** | Матрица: номенклатура (вертикаль) × города (горизонталь) × бренды по городам. Данные из таблиц «Отгрузка 1» и «Выгрузка 2» (загрузка Excel или API Google). Дашборд: план/факт за месяц, динамика, графики; лист «Лидеры» (сортировка по % выполнения); «План по брендам» (СУММЕСЛИ по брендам, факт из отгрузки). | Мок структуры и диапазонов; реальные диапазоны после выгрузки из вашей книги. |
| **8. Остатки по контрагентам** | Загрузка реализации по контрагентам по дням/месяцам (2021–2025 или скользящие 12 мес). Расчёт: разница (месяц N − месяц N+1) как средние продажи в месяц; при необходимости срез по 2 недели для затрачиваемости. Итог: средний остаток по контрагент/адрес/номенклатура. Второй источник — факт от ТП (таблица «Заявка фармацевты», формат после описания столбцов). | Мок реализации по 2–3 контрагентам; логика на тесте; формат заявки фармацевтов — после вашего описания. |
| **9. Карта** | Уже есть страница «Карта» (iframe Google My Maps). Дополнительно: слой точек из БД (адреса из справочника + расчётные остатки или факт от ТП). Переключатель: по деньгам / по регионам / по ТП / по брендам. | Реальные координаты из Адреса (2).xlsx; объёмы — из расчёта остатков (мок до загрузки данных). |
| **10. Оста по складам** | Загрузка листа «Оста» (склад, номенклатура, остаток, дата). API и экран: остатки на дату, при необходимости ежедневное обновление. | Формат столбцов по вашему отчёту 1С; при отсутствии — мок. |
| **11. Дебиторка (1210)** | Загрузка отчёта по ОСВ 1210 (по городам или по контрагенту). Список с комментариями ответственных. | Мок; реальный формат после выгрузки Дзад/отчёта. |
| **12. ОС и бухгалтерия** | Загрузка отчёта по амортизации (квартально). Реестр движений ДС и проведённых платежей (загрузка или ручной ввод). | Мок структуры; заявки на оплату и согласования — уже в ТЗ, реализую по этапам. |
| **13. ТП и маршруты** | Справочник маршрутов (месяц, ТП, список точек/адресов). Расчёт «рутов» из реализации ТП по точкам за прошлый месяц. Экран карты: выделение района/точек и экспорт маршрута для выбранного ТП. | Мок реализации по ТП; формат точек — из контрагентов/адресов. |
| **14. Логирование и умная загрузка** | Лог действий (кто, когда, что изменил) в БД; при загрузке файла — проверка «уже есть такие данные» и вывод совпадений. | Реализую с первого этапа для новых сущностей. |

Тест собираю локально: после каждого этапа проверяю сценарии (загрузка, расчёт, отображение), при отсутствии данных использую моки и помечаю в коде/конфиге «mock».

## 2. Открытые вопросы (ещё не найдены ответы)

Ниже список вопросов, от которых зависит полная или точная реализация. Ответы можно давать по мере возможности; где есть мок — тест уже можно собирать.

### ПНЛ

1. **Формулы листа «объединение данных»**  
   Какие именно ячейки или диапазоны в листе «объединение данных» ссылаются на данные 1010 и 1030? Как из них считается «разница» (пошагово или формула)? Как разница связывается с листом «свод» (номера строк/столбцов или имена диапазонов)?

2. **Структура листа «свод»**  
   Список статей ПНЛ (названия строк) и в каких столбцах/ячейках лежат план и факт по месяцам (или по одному месяцу с копированием на год).

3. **Вставка БДР в январь**  
   Точный диапазон ячеек (например, A1:F50) или имя диапазона в листе января, куда вставляются данные БДР формата бюджета, и в каком порядке столбцов (период, статья, план и т.д.).

4. **Отдельно Медхаус / Свисс**  
   В одной книге ПНЛ — оба юрлица на разных листах или один лист с колонкой «организация»? Как при объединении двух организаций вы хотите видеть свод (отдельный лист, отдельный файл)?

### Мотивация (MX)

5. **Столбцы листа «Реал 8 мес»**  
   Точные буквы/номера столбцов для: сумма с возвратами (G?), средняя реализация, процент/коэф (J?), «что вставить» (Q?), «вставленное значение» (R?), связь с планом (какой столбец соответствует плану из «Продажи дистр»).

6. **Столбцы листа «Продажи дистр»**  
   Где план на период (X?) и где средняя реализация; как именно «Реал 8 мес» и «Продажи дистр» связаны (по каким ключам — город, номенклатура, сотрудник).

7. **Связь мотивации по городам/сотрудникам**  
   На каком листе и в каких столбцах хранится разбивка плана по городам и по сотрудникам (ФИО/код), чтобы воспроизвести «раскидку» одним коэф.

8. **Формат дзад**  
   Имена столбцов листа Дзад: город, факт задолженности, план (ручной), коэффициент или иные поля для выравнивания факта к плану.

### Результаты продаж

9. **Имена листов и диапазоны**  
   Точные имена листов для: матрицы (номенклатура × города × бренды), «Отгрузка 1», «Выгрузка 2», «Лидеры», «План по брендам». Для каждого — диапазон данных (например, A1:Z100) и строка заголовков.

10. **ВПР/СУММЕСЛИ**  
    В какой таблице ключ для ВПР (номенклатура/код), в какой — суммы для СУММЕСЛИ (лист отгрузки, столбцы с брендом и суммой).

### Остатки и карта

11. **Форма «Заявка фармацевты»**  
    Список столбцов формы по ссылке (или скрин шапки таблицы): контрагент, адрес, номенклатура, остаток на дату, дата визита, ТП и т.д. — чтобы сделать одинаковый формат загрузки и якорный расчёт.

12. **Реализация по дням 2021–2025**  
    В каких файлах/листах лежит реализация по контрагентам по дням (или по месяцам): имена файлов, листы, столбцы (контрагент, дата, сумма/кол-во, номенклатура при наличии).

13. **Адреса и координаты**  
    В «Адреса (2).xlsx» и «Контрагент данные.xlsx»: в каких столбцах адрес, широта, долгота и код контрагента (или ИНН) для привязки к одной точке на карте.

### Касса, банк, оста, дебиторка

14. **Формат выгрузки 1С для кассы и банка**  
    При выгрузке в Excel: имена листов, строка заголовков и названия столбцов (дата, организация, счёт, сумма, контрагент и т.д.).

15. **Лист «Оста»**  
    Имена столбцов выгрузки остатков по складам (склад, город, номенклатура, остаток, единица, дата на которую остаток).

16. **Дебиторка (1210)**  
    Имя отчёта 1С или вид выгрузки (по городам / по контрагентам) и перечень столбцов (контрагент, сумма, срок просрочки, город и т.д.).

### Прочее

17. **Иерархия (города, сотрудники, ТП)**  
    Справочник: один файл/лист с колонками город, филиал, ФИО, роль (ТП, супервайзер), привязка к городу — или это уже есть в 1С и в каком виде выгружается.

18. **Методика KPI по торговым**  
    Формула в одном предложении или таблице: как из плана и факта (в тг или %) считается KPI/премия (например, % выполнения × базовая ставка).

19. **Денежные дни и заявки на оплату**  
    Подтверждение: оплаты только вт/чт до 16:00; список статусов заявки (черновик, на согласовании у бухгалтера, на согласовании у фин. директора, оплачено, отклонено). Нужны ли ещё статусы?

20. **Power BI и книга МХ формат**  
    Если дадите доступ на просмотр (или экспорт структуры листов/диапазонов) книги МХ формат в Google и при необходимости .pbix — смогу перенести все зависимости в документ «Источники данных» и выровнять тестовую логику с вашими листами.

## 3. Итог

- **Тест у себя** буду строить по этапам 1–14: справочники и БД → курсы и правки → ПНЛ (упрощённо) → касса/банк → мотивация (новая модель) → дзад → результаты продаж → остатки и карта → оста, дебиторка → ОС/бух → ТП/маршруты, с логированием и умной загрузкой. Всё, чего не хватает по данным или формулам, делаю на моках и помечаю.
- **Открытые вопросы** — 20 пунктов выше; по мере ваших ответов буду подставлять реальные диапазоны, форматы и формулы в [ИСТОЧНИКИ_ДАННЫХ_И_ЛОГИКА_РАСЧЕТОВ.md](../02_источники_данных/ИСТОЧНИКИ_ДАННЫХ_И_ЛОГИКА_РАСЧЕТОВ.md) и убирать моки в коде.

Если появятся новые вопросы в процессе реализации — добавлю их в этот документ в раздел «Открытые вопросы» с номером 21 и далее.
