# Деплой: что сделать тебе один раз и что буду делать я сам

## Минимум, чтобы я мог всё делать сам (без интерактива)

**Одно действие с твоей стороны:**

В любом терминале выполни один раз:

```bash
railway login
```

Откроется браузер — войди в аккаунт Railway. После этого CLI сохранит сессию, и я смогу из своей среды выполнять `railway list`, `railway link --project ID --environment ID` и дальше деплой без твоего участия.

**Если хочешь вообще без браузера:** создай [Account Token](https://railway.app/account/tokens) в Railway, пропиши его в системную переменную **RAILWAY_API_TOKEN** (Параметры системы → переменные среды) и пришли мне **Project ID** и **Environment ID** (в дашборде Railway: URL проекта вида `…/project/abc123-def456` или в настройках проекта). Тогда выполню неинтерактивно: `railway link --project <ID> --environment <ID>`.

После того как привязка будет (либо через login + я сделаю link по ID, либо ты один раз `railway link` вручную), всё остальное делаю я: деплой, переменные, домен, пуш, Vercel.

## Что нужно сделать тебе один раз (если не сделал минимум выше)

### 1. Railway: привязать проект

В терминале (в папке проекта или в `backend`):

```bash
cd "d:\Разработка медхаус\backend"
railway link
```

В интерактиве выбрать:
- проект (твой Railway-проект с MedhouseWeb);
- окружение (например production).

Либо неинтерактивно (если знаешь ID):

```bash
railway link --project <PROJECT_ID> --environment <ENVIRONMENT_ID>
```

Связка сохраняется в `.railway/` в корне проекта (эту папку лучше добавить в `.gitignore`).

**Если проект ещё не создан в Railway:** зайди на [railway.app](https://railway.app) → New Project → Deploy from GitHub repo → выбери репозиторий MedhouseWeb. В настройках сервиса укажи **Root Directory** = `backend`. Потом выполни `railway link` в папке `backend` и выбери этот проект.

---

### 2. Railway: переменные (если ещё не заданы)

В панели Railway → сервис бэкенда → **Variables** задать (или я задам через CLI после привязки):

- **SECRET_KEY** — случайная строка (`openssl rand -hex 32` или аналог).
- **DATABASE_URL** — либо Reference от сервиса Postgres (если добавил Postgres в проект), либо строка подключения к Supabase pooler (порт 6543).
- **CORS_ORIGINS** — URL фронта на Vercel, например `https://medhouse-web.vercel.app` (без слеша в конце).

Если используешь Railway Postgres: в проекте Railway добавь сервис Postgres и в сервисе бэкенда добавь переменную **Reference** `DATABASE_URL` из этого Postgres.

---

### 3. Vercel: проект и настройки

- Импортируй репозиторий MedhouseWeb на [vercel.com](https://vercel.com), если ещё не импортирован.
- **Root Directory** = `frontend`, **Framework** = Vite.
- В **Environment Variables** добавь **VITE_API_BASE** = URL бэкенда на Railway (без слеша в конце), например `https://medhouse-web-production.up.railway.app`. (Значение можно подставить после того, как у бэкенда будет домен.)

---

### 4. Один раз инициализировать БД

Если БД пустая (новый Postgres или новый проект):

- Локально: в `backend/.env` задать `DATABASE_URL` строкой от Railway/Supabase и выполнить `cd backend && python -m scripts.init_db` (и при необходимости `python -m scripts.create_dashboard_templates`).
- Или через Railway: Run Command / one-off с командой `python -m scripts.init_db` (если такой скрипт есть в репо).

Это нужно сделать один раз после первого деплоя бэкенда.

## Что буду делать я сам (после твоих шагов выше)

- **Коммит и пуш** — все изменения в коде (конфиг, фронт, доки) коммичу и пушу в `origin master`.
- **Railway: деплой** — из папки `backend` выполняю `railway up` (или через MCP deploy с нужным `workspacePath`).
- **Railway: переменные** — при необходимости задаю через `railway variable set KEY=value`.
- **Railway: домен** — при необходимости вызываю `railway domain` и подсказываю тебе URL для `VITE_API_BASE` и `CORS_ORIGINS`.
- **Railway: логи** — по запросу смотрю `railway logs` и помогаю разобрать ошибки.
- **Vercel** — пуш в GitHub при включённой интеграции сам запускает деплой; при необходимости вызываю MCP `deploy_to_vercel`.
- **Supabase** — через MCP смотрю таблицы, при необходимости подсказываю, что проверить в дашборде или в коде.

Как только ты сделаешь п. 1 (и при необходимости 2–4), можешь написать «деплой» или «задеплой» — сделаю коммит, пуш и деплой на Railway (и подскажу по Vercel), и отпишу лог действий.
