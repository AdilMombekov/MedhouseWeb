# Начните здесь

**Если вы новый человек в проекте** (разработчик, аналитик, подрядчик) — этот файл ваш главный вход. По нему можно разобраться в проекте и начать работу **без автора инструкций**. Всё, что нужно, разложено по папкам ниже.

---

## Что это за проект

- **Название:** Medhouse / Swiss Energy — управленческая отчётность.
- **Суть:** веб-сайт (дашборд) для двух компаний: **Медхаус** (дистрибьютор, 15 городов) и **Свисс Энерджи** (контролирующая компания). На сайте: ПНЛ, мотивация, касса/банк, результаты продаж, остатки, карта, дебиторка, бухгалтерия и др.
- **Стек:** backend — FastAPI (Python), frontend — React + Vite; БД — SQLite (для прода при росте — PostgreSQL). Данные берутся из папки «база для данных», из Google Drive (ПНЛ, МХ формат, папки с форматом бюджета) и из выгрузок 1С.

## Структура документации (папки)

Всё лежит в папке **`docs/`**. Зачем каждая папка:

| Папка | Назначение | Когда открывать |
|-------|------------|-----------------|
| **01_инструктаж** | Пошаговый инструктаж для нового сотрудника: что прочитать, в каком порядке, что настроить, как проверить. | В первый день; если что-то непонятно в «что делать». |
| **02_источники_данных** | Откуда брать данные (файлы, листы, ячейки), в каком формате, логика расчётов по каждому блоку (ПНЛ, мотивация, касса/банк, остатки, карта и т.д.). Список требуемых данных и описание папки «база для данных». | Когда нужно реализовать загрузку или расчёт; когда нужно понять формат файлов и таблиц. |
| **03_архитектура_и_тз** | Архитектура продукта, полное ТЗ, масштаб, безопасность. План тестовой реализации по этапам и список открытых вопросов (на что пока нет ответа). | Когда нужно понять общую картину, этапы разработки и что ещё нужно уточнить. |
| **04_настройка_и_пути** | Пути к данным, переменные окружения, проверка путей. **Полный список требований:** [ЧТО_ТРЕБУЕТСЯ_БЭК_БД_БЭКАП.md](04_настройка_и_пути/ЧТО_ТРЕБУЕТСЯ_БЭК_БД_БЭКАП.md) — бэкенд, БД, суббазы, бэкап, окружение. | При первом запуске, настройке сервера и при добавлении новых источников. |

## Можете ли вы сделать всё без автора инструкций?

- **Да, если:** вы выполняете шаги 1–7 выше, читаете документы в указанных папках и используете план реализации (этапы + моки там, где данных ещё нет). Открытые вопросы в документе «ПЛАН_ТЕСТОВОЙ_РЕАЛИЗАЦИИ_И_ОТКРЫТЫЕ_ВОПРОСЫ» помечают, что нужно уточнить у заказчика/автора; пока ответов нет — в коде используются моки или заглушки.
- **Нет, только если:** нужны решения по бизнес-логике, которых нет в документах (например, новая формула ПНЛ или новый источник данных). Тогда в «Открытые вопросы» или в «Источники данных» нужно добавить пункт и по возможности получить ответ от автора инструкций или заказчика.
