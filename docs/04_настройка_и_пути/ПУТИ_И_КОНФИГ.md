# Пути и конфигурация

Краткая сводка: где лежат данные и какие переменные окружения задают пути. Этого достаточно, чтобы настроить окружение и проверить доступ **без автора инструкций**.

---

## 1. Локальная папка «база для данных»

- **Назначение:** основные файлы-источники для отчётности (ПНЛ, справочники, адреса).
- **Где задаётся путь:** в коде — `backend/app/config.py`, переменная **`DATA_BASE_DIR`**. По умолчанию: **`<корень проекта>/база для данных`** (рядом с папками `backend`, `frontend`, `docs`).
- **Как задать другой путь:** в файле `backend/.env` добавьте переменную **`DATA_BASE_PATH`** и укажите полный путь к папке, например:  
  `DATA_BASE_PATH=D:\Мои данные\база для данных`
- **Что должно лежать в папке:** файлы ОТЧЕТ МХК 2026.xlsx, Номенклатура.xlsx, Контрагент данные.xlsx, Адреса (2).xlsx; при наличии — внешняя обработка 1С (.epf) и отчёт Power BI (.pbix). Подробнее: [02_источники_данных/БАЗА_ДАННЫХ_ОПИСАНИЕ.md](../02_источники_данных/БАЗА_ДАННЫХ_ОПИСАНИЕ.md).

## 3. Проверка путей

Из папки **backend** выполните:

```bash
cd backend
python -m scripts.check_paths
```

Скрипт выведет:
- содержимое папки «база для данных» (наличие ожидаемых файлов и .epf);
- при настроенном сервисном аккаунте — список файлов в папке ПНЛ и при заданных ID — в папках …Евгений и …Алмас;
- ID таблицы МХ и заявки фармацевты.

Если папка «база для данных» не найдена — проверьте `DATA_BASE_PATH` в `.env` или создайте папку в корне проекта.

## 4. Остальные переменные окружения (кратко)

- **SECRET_KEY** — секрет для JWT (обязательно сменить в продакшене).
- **DATABASE_URL** — по умолчанию `sqlite:///./medhouse.db`.
- **MEDHOUSE_API_KEY_1**, **_2**, **_3** — API-ключи для доступа из внешних программ (см. README и API_ACCESS.md).
- Полный список с комментариями: **backend/.env.example**.
