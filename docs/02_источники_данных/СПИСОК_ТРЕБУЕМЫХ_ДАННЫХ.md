# Список данных, которые требуются от вас

Ниже перечислено, **какие данные и в каком виде нужны** для формирования отчётности на сайте. Часть уже есть в папке **«база для данных»** (см. [БАЗА_ДАННЫХ_ОПИСАНИЕ.md](БАЗА_ДАННЫХ_ОПИСАНИЕ.md)); остальное — нужно подготовить или уточнить.

## Уже в папке «база для данных» (обработано)

| Файл | Назначение | Что использовать |
|------|------------|------------------|
| **ОТЧЕТ МХК 2026.xlsx** | Отчёт ПНЛ/данные по МХК за 2026 г. | Листы: ПНЛ (период, параметры), Номенклатура.Номенклатура (17 столбцов), LOT (лоты). Основа для факта ПНЛ и справочников. |
| **Номенклатура.xlsx** | Справочник номенклатуры | Лист «Номенклатура.Номенклатура» — код, наименование и др. Единый справочник для остатков, мотивации, карты. |
| **Контрагент данные.xlsx** | Данные по контрагентам | Листы «Контрагент_1» — контрагент, код, ИНН/БИН, наименование, даты, суммы, Profit. Для карты, остатков, дебиторки, ABC. |
| **Адреса (2).xlsx** | Адреса (контрагенты/точки) | Листы с адресами, привязка к контрагентам. Для карты и географии. |

## 1. ПНЛ (уровень 1 — Руководство)

| Что нужно | Формат | Где взять / примечание |
|-----------|--------|------------------------|
| **Факт ПНЛ по месяцам** | В книге ПНЛ в Google Drive по каждому месяцу — лист **«объединение данных»**: туда тянутся счета **1010** (касса) и **1030** (банк) из 1С + организация в ОСВ. Из объединения считается **разница**, она раскидывается по показателям; статьи — из листа **«свод»**. | Нужна выгрузка формул/структуры листа «объединение данных» (какие ячейки — 1010/1030, разница, связь с сводом). |
| **Планы (бюджет на год)** | В **левом конце листа января** вставка данных **БДР формата бюджета**. Формат лежит в папках Drive, в названии которых в конце **Евгений** (Медхаус) или **Алмас** (Свисс). | ID папок …Евгений и …Алмас в .env; имена файлов БДР и ячейки вставки в январе. |
| **Свод (форма)** | Лист «свод» в той же книге ПНЛ; первозданный вид формы. | ID книги, имя листа, диапазоны для заливки план/факт. |
| **Курсы валют по месяцам** | Месяц, курс (ручная корректировка). | Сайт/БД или начальная выгрузка. |

| **Прайс-лист** | Номенклатура, цена, валюта, период действия (если есть). | Файл/лист в «база для данных» или Google; имена и столбцы. |
| **Планы по отгрузкам/возвратам (на месяц)** | Город/ТП, номенклатура или группа, план (шт/тг). Обновление до 10 числа. | Excel или таблица на сайте; при наличии — имя файла и лист. |

## 3. Касса, Банк

| Что нужно | Формат | Где взять / примечание |
|-----------|--------|------------------------|
| **Касса** | Счёт **1010** + организация в **ОСВ** (1С). В МХ формат — лист **«Касса»**: отчёт из 1С вставляют с 1 по текущее число каждый день с утра. | Зафиксировано в ИСТОЧНИКИ_ДАННЫХ. |
| **Банк** | Счёт **1030** + организация в ОСВ. В МХ формат — лист **«Банк»**: ежедневная вставка с 1 по текущую дату. | Зафиксировано в ИСТОЧНИКИ_ДАННЫХ. |

## Power BI

Файл **Power BI** (`.pbix`) в папке проекта не найден. Чтобы описать ваши наработки и согласовать логику с сайтом:

- Положите файл `.pbix` в папку **«база для данных»** или в корень проекта, либо  
- Укажите **полный путь** к файлу на вашем компьютере — тогда можно будет описать структуру данных и визуалы (по имени и типичным связям Power BI), без открытия самого .pbix в среде.

После появления файла или пути добавлю в описание раздел «Наработки Power BI» и связь с перечнем требуемых данных выше.

## Итог: что сделать в первую очередь

1. **ПНЛ:** подтвердить, что факт по Медхаусу берётся из «ОТЧЕТ МХК 2026.xlsx» и/или из внешней обработки 1С; прислать описание обработки (имя, откуда какие строки) или положить .epf в «база для данных».  
2. **Планы и свод:** точные имена файлов бюджета в папках Евгений/Алмас, ID и лист «свод» в Google Drive, список ячеек для заливки.  
3. **Мотивация:** имена файлов/листов и перечень столбцов для реал 8 мес, дзад, остаток, прайс.  
4. **Касса/Банк:** имена отчётов 1С и при выгрузке в Excel — имена файлов и листов.  
5. **Карта:** в каких столбцах в «Адреса (2).xlsx» и «Контрагент данные.xlsx» лежат адрес и при наличии — широта/долгота.  
6. **Power BI:** положить .pbix в «база для данных» или указать путь к нему.

По мере поступления данных этот список можно помечать (например, «получено») и переносить уточнения в [ИСТОЧНИКИ_ДАННЫХ_И_ЛОГИКА_РАСЧЕТОВ.md](ИСТОЧНИКИ_ДАННЫХ_И_ЛОГИКА_РАСЧЕТОВ.md).
