# Папка «база для данных» — описание и использование

В папке **«база для данных»** собраны файлы-источники для формирования отчётности. Ниже — что уже лежит в папке, как это связано с разделами сайта и что нужно добавить (в т.ч. внешнюю обработку 1С для ПНЛ).

---

## Расположение и проверка по путям

Путь к папке задаётся в проекте: **`backend/app/config.py`** → `DATA_BASE_DIR`. По умолчанию: **`<корень проекта>/база для данных`** (рядом с папками `backend`, `frontend`, `docs`). Переопределение: переменная окружения **`DATA_BASE_PATH`** в `backend/.env`.

**Проверка по всем указанным вами путям (локальная папка + Google Drive):**
```bash
cd backend
python -m scripts.check_paths
```
Скрипт выводит: содержимое папки «база для данных» (наличие ОТЧЕТ МХК 2026.xlsx, Номенклатура.xlsx, Контрагент данные.xlsx, Адреса (2).xlsx, .epf); при настроенном сервисном аккаунте — список файлов в папке ПНЛ и при заданных ID — в папках Евгений и Алмас.

## Файлы в папке (текущее состояние)

### 1. ОТЧЕТ МХК 2026.xlsx

| Параметр | Значение |
|----------|----------|
| Назначение | Отчёт по МХК (Медхаус) за 2026 год; основа для **факта ПНЛ** и связанных справочников. |
| Листы | **ПНЛ** — параметры отчёта (период с 2026-01-01 по 2026-12-31), служебные ячейки. **Лист16** — служебный (1 столбец). **Номенклатура.Номенклатура** — до 17 столбцов (код, наименование и др.). **LOT** — лоты (номенклатура, дата поступления, LOT и др., 8 столбцов). |
| Использование | Период и структура ПНЛ — для формирования отчёта «Уровень 1 — Руководство». Номенклатура и LOT — справочники для остатков, мотивации, себестоимости. |
| Связь с 1С | Файл, скорее всего, формируется внешней обработкой 1С (ПНЛ). Нужно: описание обработки или сама обработка (.epf) в этой папке, чтобы зафиксировать, из каких документов/регистров берутся строки факта. |

**Что уточнить:** какие именно столбцы на листе «ПНЛ» соответствуют статьям ПНЛ и суммам по месяцам; при необходимости — выгрузка по месяцам в отдельные столбцы/листы.

## Внешняя обработка 1С для ПНЛ

Вы указали, что добавили в папку **внешнюю обработку для построения ПНЛ**. В репозитории в папке «база для данных» файлы с расширением **.epf** не обнаружены. Возможные варианты:

1. Обработка лежит в другом каталоге — тогда скопируйте её в **«база для данных»** (например, `ПНЛ_внешняя.epf` или как у вас называется).
2. Обработка выгружает результат в Excel (например, «ОТЧЕТ МХК 2026.xlsx») — тогда достаточно описать в одном абзаце или в таблице: **имя обработки**, **какие документы/регистры 1С** она использует для каждой строки/статьи ПНЛ. Это можно добавить в [ИСТОЧНИКИ_ДАННЫХ_И_ЛОГИКА_РАСЧЕТОВ.md](ИСТОЧНИКИ_ДАННЫХ_И_ЛОГИКА_РАСЧЕТОВ.md) и в [СПИСОК_ТРЕБУЕМЫХ_ДАННЫХ.md](СПИСОК_ТРЕБУЕМЫХ_ДАННЫХ.md).

После появления .epf или описания источников ПНЛ в 1С — обновлю этот раздел и список требуемых данных.

- **Ссылка:** [Google Таблица](https://docs.google.com/spreadsheets/d/1XSFq5PbJadJR21h6Z9z3hhfnGu4tEl93K6WEXUYutdI/edit?gid=1144561640#gid=1144561640) (лист gid=1144561640). Доступ по авторизации; структуру столбцов при возможности описать здесь или в ИСТОЧНИКИ_ДАННЫХ_И_ЛОГИКА_РАСЧЕТОВ.md.
- **Бот:** фактические остатки от ТП собираются ботом (D: dev bot1 фармаси), данные — в таблицу «Заявка фармацевты».
