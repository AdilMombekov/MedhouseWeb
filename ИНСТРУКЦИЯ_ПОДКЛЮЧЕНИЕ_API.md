# Подключение к API по ключу (3 ключа)

В проекте настроены **3 API-ключа**. Любой из них даёт полный доступ к API (уровень администратора). Ключи лежат в файле **`backend/.env`** (переменные `MEDHOUSE_API_KEY_1`, `MEDHOUSE_API_KEY_2`, `MEDHOUSE_API_KEY_3`).

---

## Шаг 1. Узнать ключи

Откройте файл **`backend/.env`** и скопируйте значение одной из переменных:

- `MEDHOUSE_API_KEY_1=...`
- `MEDHOUSE_API_KEY_2=...`
- `MEDHOUSE_API_KEY_3=...`

Скопируйте только **значение после `=`** (без пробелов).

---

## Шаг 2. Перезапустить бэкенд

Чтобы ключи из `.env` подхватились:

```bash
cd backend
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

(Если бэкенд уже запущен с `--reload`, он перезапустится сам после сохранения `.env`.)

---

## Шаг 3. Подключение

### Вариант A: заголовок X-API-Key

В каждой программе добавьте заголовок:

| Имя заголовка | Значение        |
|---------------|------------------|
| `X-API-Key`   | один из 3 ключей |

**curl:**

```bash
curl -H "X-API-Key: ВСТАВЬТЕ_СЮДА_КЛЮЧ_ИЗ_ENV" http://localhost:8000/api/auth/me
```

**Postman:**

1. Откройте запрос.
2. Вкладка **Headers** → Key: `X-API-Key`, Value: вставьте ключ из `.env`.
3. URL, например: `http://localhost:8000/api/auth/me` или `http://localhost:8000/api/analytics/bootstrap`.

### Вариант B: Authorization Bearer

Вместо заголовка `X-API-Key` можно передать ключ как Bearer-токен:

- **Header:** `Authorization: Bearer ВАШ_КЛЮЧ`

**Postman:** вкладка **Authorization** → Type: **Bearer Token** → Token: вставьте ключ.

---

## Шаг 4. Проверка

Запрос без ключа должен вернуть **401**. С ключом — **200** и JSON.

**Проверка доступа (с ключом):**

```bash
curl -H "X-API-Key: ВАШ_КЛЮЧ" http://localhost:8000/api/auth/me
```

Ожидается ответ с полями `id`, `email`, `role` (например `"role": "admin"`).

**Список эндпоинтов:** в браузере откройте **http://localhost:8000/docs** и вызывайте любые методы, добавляя вручную заголовок `X-API-Key` в разделе "Authorize" (если Swagger поддерживает) или тестируйте через Postman/curl.

## Полезные URL (база: http://localhost:8000)

| Метод | URL | Описание |
|-------|-----|----------|
| GET | /api/auth/api-key-info | Как подключаться по ключу (без авторизации) |
| GET | /api/auth/me | Текущий пользователь (проверка ключа) |
| GET | /api/analytics/bootstrap | Данные для главной страницы дашборда |
| GET | /api/companies | Список компаний |
| GET | /api/templates | Список шаблонов |
| GET | /api/uploads | Загруженные отчёты |
| GET | /api/users | Список пользователей (admin) |

---

## Безопасность

- Файл **`backend/.env`** не должен попадать в Git (он в `.gitignore`). Не публикуйте ключи.
- Для продакшена лучше сгенерировать новые ключи и прописать их только в переменных окружения на сервере.
- При утечке ключа удалите его из `.env` и перезапустите бэкенд; при необходимости добавьте новый ключ через `MEDHOUSE_API_KEY_1` / `_2` / `_3`.
