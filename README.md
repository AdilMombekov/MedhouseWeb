# Medhouse / Swiss Energy — Управленческая отчётность

Дашборд и бэкенд для управленческой отчётности двух компаний:
- **Медхаус** — дистрибьютор парафармации, 15 городов и регионов.
- **Свисс Энерджи** — контролирующая компания Swiss Group (Швейцария), производство в UA/BG/CN, поставки в филиалы (в т.ч. Медхаус через оформление в Казахстане).

## Возможности

- **Дашборд** — аналитика с акцентом на папку **Анализ 2** (годовые файлы 2020–2025, логистика), превью таблиц, агрегаты по годам.
- **Импорт отчётов** — загрузка XLSX/XLS/CSV от операторов (отчёты из 1С по шаблону).
- **Шаблоны** — скачивание шаблонной формы для заполнения в 1С и последующей загрузки.
- **Роли**: Администратор, Модератор, Оператор. Админы управляют пользователями; модераторы и админы загружают отчёты.
- **Интеграция Google Drive** — опционально: чтение файлов из ваших папок/документов после настройки OAuth (см. [GOOGLE_DRIVE.md](GOOGLE_DRIVE.md)).
- **Рассылка из 1С** — автоматическая выгрузка «Выгрузка XLSX (2).xlsx» импортируется в БД и доступна по API; структура и логирование описаны в [РАССЫЛКА.md](РАССЫЛКА.md).
- **Google Таблицы** — все таблицы, доступные сервисному аккаунту (или OAuth), можно обнаружить и залогировать в БД командой `python -m scripts.sync_google_sheets`; чтение через API `/api/google-sheets/*` (см. [GOOGLE_DRIVE.md](GOOGLE_DRIVE.md)).
- **База 1С** — расположение и состав базы Medh_2026-02-24 в `D:\1с` описаны в [1C_БАЗА.md](1C_БАЗА.md) (файловая база, DT, связь с выгрузками и проектом).

## Запуск

### Бэкенд (FastAPI)

```bash
cd backend
python -m venv venv
venv\Scripts\activate   # Windows
pip install -r requirements.txt
python -m scripts.init_db
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

После `init_db` создаётся админ: **admin@medhouse.kz** / **admin123**.

### Фронтенд (React + Vite)

```bash
cd frontend
npm install
npm run dev
```

Откройте http://localhost:3030. API проксируется на http://127.0.0.1:8000.

## Структура данных

- Папка **«база для данных»** в корне проекта — централизованные источники: «ОТЧЕТ МХК 2026.xlsx», «Номенклатура.xlsx», «Контрагент данные.xlsx», «Адреса (2).xlsx»; сюда же — внешняя обработка 1С (.epf) и при необходимости Power BI (.pbix). Пути (в т.ч. Google Drive: ПНЛ, Евгений, Алмас, МХ формат, заявка фармацевты) заданы в `backend/app/config.py` и `backend/.env`. **Проверка по путям:** `cd backend && python -m scripts.check_paths`. Подробнее: [docs/02_источники_данных/БАЗА_ДАННЫХ_ОПИСАНИЕ.md](docs/02_источники_данных/БАЗА_ДАННЫХ_ОПИСАНИЕ.md).
- Папка **Анализ 2** в корне проекта — основные данные для аналитики (годы 2020–2025, логистика). Остальные файлы в корне — референсы.
- Файл **Выгрузка XLSX (2).xlsx** (рассылка из 1С) — после запуска `python -m scripts.import_mailing_to_db` данные попадают в таблицы `mailing_batches` и `mailing_rows` (см. РАССЫЛКА.md).
- Загруженные отчёты сохраняются в `backend/uploads/`.
- Шаблоны лежат в `backend/templates/`; при первом запуске `init_db` создаётся пример шаблона.

## Переменные окружения (backend)

Скопируйте `backend/.env.example` в `backend/.env` и при необходимости задайте:

- `SECRET_KEY` — секрет для JWT.
- `DATABASE_URL` — по умолчанию SQLite `./medhouse.db`; для прода — Postgres (например из Supabase, см. DEPLOY.md).
- `LOG_LEVEL` — уровень логов: DEBUG, INFO, WARNING, ERROR (по умолчанию INFO). `LOG_FILE` — путь к файлу логов (опционально).
- **`MEDHOUSE_API_KEY_1`, _2, _3`** — три API-ключа для доступа из Postman, curl и сторонних программ (полный доступ как admin). Ключи уже сгенерированы и записаны в `backend/.env`. Инструкция по подключению: [ИНСТРУКЦИЯ_ПОДКЛЮЧЕНИЕ_API.md](ИНСТРУКЦИЯ_ПОДКЛЮЧЕНИЕ_API.md). Подробнее: [API_ACCESS.md](API_ACCESS.md).
- Для Google Drive используйте проект **pharmacy-bot-487900** (Google Cloud), положите `credentials.json` в папку `backend/` и при первом запросе пройдите OAuth (см. GOOGLE_DRIVE.md).
