# Доступ к API по ключу (Postman, curl, скрипты)

Для проверки API сторонними программами без входа через браузер используются **API-ключи**. Каждый ключ даёт полный доступ к API (уровень администратора). В проекте настроены **3 ключа** (переменные `MEDHOUSE_API_KEY_1`, `MEDHOUSE_API_KEY_2`, `MEDHOUSE_API_KEY_3` в `backend/.env`).

**Краткая инструкция:** [ИНСТРУКЦИЯ_ПОДКЛЮЧЕНИЕ_API.md](ИНСТРУКЦИЯ_ПОДКЛЮЧЕНИЕ_API.md)

---

## 1. Получить API-ключ

Ключи задаются в `backend/.env` (уже добавлены 3 ключа). Либо сгенерируйте новые и подставьте в `.env`:

### Сгенерировать новый ключ

```bash
cd backend
python -m scripts.generate_api_key
```

Добавьте вывод в `backend/.env` как `MEDHOUSE_API_KEY_1=...` (или _2, _3). Перезапустите бэкенд.

### Узнать, настроен ли ключ (без показа самого ключа)

В браузере или curl без авторизации:

```bash
curl http://localhost:8000/api/auth/api-key-info
```

Там же будут примеры использования.

---

## 2. Как передавать ключ

Доступны два способа (на выбор).

### Вариант A: заголовок `X-API-Key`

Добавьте заголовок с именем **`X-API-Key`** и значением — ваш ключ.

**curl:**

```bash
curl -H "X-API-Key: ВАШ_КЛЮЧ" http://localhost:8000/api/auth/me
```

**Postman:**

1. Запрос → вкладка **Headers**.
2. Key: `X-API-Key`, Value: ваш API-ключ.

### Вариант B: заголовок `Authorization: Bearer`

Вместо JWT можно передать сам API-ключ как Bearer-токен.

**curl:**

```bash
curl -H "Authorization: Bearer ВАШ_КЛЮЧ" http://localhost:8000/api/auth/me
```

**Postman:**

1. Запрос → вкладка **Authorization**.
2. Type: **Bearer Token**.
3. Token: вставьте ваш API-ключ.

---

## 3. Базовый URL и примеры эндпоинтов

- Локально: **http://localhost:8000** (или **http://127.0.0.1:8000**).
- Документация: http://localhost:8000/docs.

Примеры запросов с ключом (подставьте свой ключ и хост при необходимости):

```bash
# Проверка доступа (текущий «пользователь»)
curl -H "X-API-Key: ВАШ_КЛЮЧ" http://localhost:8000/api/auth/me

# Данные для главной страницы дашборда
curl -H "X-API-Key: ВАШ_КЛЮЧ" http://localhost:8000/api/analytics/bootstrap

# Список компаний
curl -H "X-API-Key: ВАШ_КЛЮЧ" http://localhost:8000/api/companies

# Список пользователей (только admin)
curl -H "X-API-Key: ВАШ_КЛЮЧ" http://localhost:8000/api/users

# Список загруженных отчётов
curl -H "X-API-Key: ВАШ_КЛЮЧ" http://localhost:8000/api/uploads
```

## 4. Postman: один раз настроить ключ

Чтобы не вставлять ключ в каждый запрос:

1. **Environments** → создать переменную, например `medhouse_api_key`, значение — ваш ключ.
2. В запросе во вкладке **Headers** добавить:
   - Key: `X-API-Key`
   - Value: `{{medhouse_api_key}}`
3. Либо в **Authorization** выбрать Bearer Token и в поле Token указать `{{medhouse_api_key}}`.

Дальше все запросы к этому API будут автоматически отправлять ключ.

---

## 5. Безопасность

- Ключ даёт **полные права администратора**. Не передавайте его по незащищённым каналам и не храните в публичных репозиториях.
- В продакшене задавайте **`MEDHOUSE_API_KEY`** только на сервере (переменная окружения), не в коде.
- При утечке сгенерируйте новый ключ и обновите его в `.env`, затем перезапустите бэкенд.
