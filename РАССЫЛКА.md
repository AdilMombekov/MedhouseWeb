# Рассылка из 1С — структура и логирование в БД

Автоматическая рассылка из 1С приходит в виде файла **«Выгрузка XLSX (2).xlsx»** (или с похожим именем). Данные из него полностью изучены и залогированы в базу проекта.

---

## Файл рассылки

- **Имя:** `Выгрузка XLSX (2).xlsx` (в корне проекта).
- **Лист:** один — `Лист_1`.
- **Размер:** ~7000 строк × 27 колонок (каждая строка — документ реализации/отгрузки).

---

## Колонки (27 полей из 1С)

| № | Колонка в файле | Описание |
|---|------------------|----------|
| 1 | Номенклатура | Наименование товара (SKU) |
| 2 | Организация | MedHouse |
| 3 | Период дата | Дата документа (например 04.03.2026) |
| 4 | Склад.Склад | Склад / город (Склад, Сетевой и т.д.) |
| 5 | Контрагент | Наименование покупателя (ТОО, ИП) |
| 6 | Номер документа движения | Номер реализации (например 77aku000216) |
| 7 | Подразделение | MedPack, HomeTest и т.д. |
| 8 | ИНН НДС | ИНН контрагента (число) |
| 9 | % НДС | Ставка НДС (5%, 12% и т.д.) |
| 10 | Валюта документа | Вал. |
| 11 | Покупатель (контрагент) | Полное описание покупателя и ссылка на документ |
| 12 | Номенклатура | Повтор/расшифровка номенклатуры |
| 13 | Адрес поставки | Адрес доставки |
| 14 | Город | Город |
| 15 | Тип документа | Реализация (товары, услуги) и т.д. |
| 16 | Валюта | KZT |
| 17 | Вид операции | Нал/Без нал и т.д. |
| 18 | Сумма | Сумма по документу |
| 19 | … | Доп. поле |
| 20 | … | Доп. поле |
| 21 | Номер контракта / ИНН | ИНН и т.д. |
| 22 | Условия оплаты | Текст (например «7% от 100% предоплаты») |
| 23 | Количество продажи (в единицах) | Кол-во единиц |
| 24 | Сумма продажи (в единицах) | Выручка в тенге |
| 25 | Номер по порядку | Порядковый номер |
| 26 | Цена за ед. | Цена единицы |
| 27 | Profit (тенге) | Прибыль в тенге |

Все 27 полей каждой строки сохраняются в БД в виде JSON в поле `raw_data`.

---

## Таблицы в БД

### `mailing_batches`

Одна запись = один загруженный файл рассылки.

| Поле | Тип | Описание |
|------|-----|----------|
| id | int | PK |
| source_file | str | Имя файла (Выгрузка XLSX (2).xlsx) |
| sheet_name | str | Имя листа |
| row_count | int | Количество импортированных строк |
| created_at | datetime | Время импорта |

### `mailing_rows`

Одна запись = одна строка из рассылки (один документ).

| Поле | Тип | Описание |
|------|-----|----------|
| id | int | PK |
| batch_id | int | FK → mailing_batches.id |
| row_index | int | Номер строки в файле |
| period_date | str | Период дата (04.03.2026) |
| organization | str | Организация (MedHouse) |
| kontragent | str | Контрагент |
| nomerklatura | str | Номенклатура |
| amount | float | Сумма продажи (в единицах) |
| profit | float | Profit (тенге) |
| raw_data | text | JSON со всеми 27 колонками строки |
| created_at | datetime | Время импорта |

## Импорт в БД

Из папки `backend`:

```bash
python -m scripts.import_mailing_to_db
```

Скрипт ищет в корне проекта файл с именем, содержащим `XLSX (2)`, создаёт новую партию (`mailing_batches`) и вставляет все строки в `mailing_rows`. При каждой новой рассылке из 1С достаточно положить новый файл в корень и снова запустить скрипт — будет создана новая партия.

---

## API (требуется авторизация)

- **GET /api/mailing/batches** — список партий рассылки (последние 50).
- **GET /api/mailing/rows** — строки рассылки с фильтрами:
  - `batch_id` — по партии;
  - `organization` — по организации;
  - `period_date` — по дате (04.03.2026);
  - `kontragent` — поиск по контрагенту (подстрока);
  - `limit`, `offset` — постраничная выборка (по умолчанию последняя партия, limit=100).
- **GET /api/mailing/stats** — сводка: число партий, число строк, последняя партия.

В ответе каждой строки возвращаются ключевые поля и полный JSON `raw_data` со всеми колонками из 1С.
