# Подключение Google Drive

Сайт **не имеет доступа** к вашему Google Drive по умолчанию. Чтобы приложение могло читать ваши файлы (управленческая отчётность по городам, центральному офису и т.д.), нужно один раз настроить OAuth в Google Cloud и положить файл учётных данных на сервер.

## Могу ли я (ИИ) подключаться к вашему Google Drive?

**Нет.** Я работаю только в вашем проекте на вашем компьютере. Я не могу зайти в ваш аккаунт Google и прочитать файлы с вашего диска.  

**Да.** Ваш **бэкенд** (сервер приложения) может читать файлы с Google Drive **после того, как вы**:

1. Создадите проект в Google Cloud и включите Google Drive API.
2. Создадите OAuth 2.0 учётные данные и скачаете файл `credentials.json`.
3. Положите `credentials.json` в папку `backend` вашего проекта.
4. Один раз при первом запросе к разделу «Google Drive» на сайте войдёте в Google в браузере и разрешите доступ приложению.

После этого **сервер** будет использовать сохранённый токен и по вашим запросам (например, по ID папки) сможет получать список файлов и скачивать их для импорта в отчётность.

## Откуда взять credentials.json

Файл **credentials.json** вы создаёте и скачиваете в Google Cloud Console:

1. Откройте: **[Учётные данные (Credentials)](https://console.cloud.google.com/apis/credentials?project=pharmacy-bot-487900)** — откроется ваш проект pharmacy-bot-487900.
2. Вверху нажмите **«+ Создать учётные данные»** → **«Идентификатор клиента OAuth»**.
3. Если появится предложение настроить экран согласия OAuth — нажмите **«Настроить экран согласия»**, выберите тип **«Внешний»**, укажите название приложения и email, сохраните.
4. Снова: **«Создать учётные данные»** → **«Идентификатор клиента OAuth»**.
5. **Тип приложения:** выберите **«Приложение для ПК»** (Desktop app).  
   **Имя:** например, `Medhouse Drive`.
6. Нажмите **«Создать»**. В списке учётных данных появится новый клиент — в столбце справа нажмите **«Скачать JSON»** (иконка со стрелкой вниз).
7. Сохраните скачанный файл и переименуйте его в **`credentials.json`**.
8. Положите **`credentials.json`** в папку **`backend`** вашего проекта (рядом с `app` и `main.py`).

После этого перезапустите бэкенд — при первом обращении к разделу Google Drive в дашборде откроется браузер для входа в Google.

## Пошаговая настройка (подробно)

### 1. Google Cloud Console

1. Откройте [Google Cloud Console](https://console.cloud.google.com).
2. Выберите ваш проект: **Project ID: pharmacy-bot-487900** (Project number: 142024412541) или создайте новый.
3. В меню **«API и сервисы»** → **«Библиотека»** найдите **Google Drive API** и нажмите **Включить**.

### 2. Экран согласия OAuth

1. **«API и сервисы»** → **«Экран согласия OAuth»**.
2. Тип приложения: **Внешний** (если только для своих сотрудников, можно потом ограничить круг пользователей).
3. Заполните название приложения и контактную почту, сохраните.

### 3. Учётные данные OAuth 2.0

1. **«API и сервисы»** → **«Учётные данные»** → **«Создать учётные данные»** → **«Идентификатор клиента OAuth»**.
2. Тип приложения: **«Приложение для ПК»** (Desktop app).
3. Имя — любое (например, «Medhouse Dashboard»).
4. Нажмите **Создать**. Скачайте JSON-файл с ключами клиента.

### 4. Установка в проект

1. Переименуйте скачанный файл в **`credentials.json`**.
2. Положите его в папку **`backend`** вашего проекта (рядом с `app/` и `main.py`).
3. Перезапустите бэкенд.

### 5. Первый вход

1. В дашборде откройте вкладку **«Google Drive»**.
2. Нажмите **«Список файлов»**, указав ID папки (или оставьте пустым для корня).
3. При первом запросе откроется браузер — войдите в нужный Google-аккаунт и разрешите доступ приложению.
4. В папке `backend` появится файл **`token.json`** — его не нужно отдавать никому и не удаляйте, иначе придётся авторизоваться снова.

## Как узнать ID папки или файла в Google Drive

- Откройте папку или файл в Google Drive в браузере.
- Ссылка будет вида:  
  `https://drive.google.com/drive/folders/**1ABC...xyz**`  
  или  
  `https://drive.google.com/file/d/**1ABC...xyz**`  
- Часть **`1ABC...xyz`** — это ID. Вставьте его в поле «ID папки на Google Drive» на вкладке Google Drive в дашборде и нажмите «Список файлов».

После настройки вы можете присылать ссылки на папки/документы с управленческой отчётностью по городам и центральному офису — в интерфейсе будет возможность выбирать эти источники и использовать данные для аналитики.
