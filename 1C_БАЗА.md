## 4. Что можно и что нельзя делать с базой программно

- **Читать напрямую .1CD и .dt** из своего кода (Python, сайт и т.д.) **нельзя**: форматы проприетарные, нет открытого API. Работа с ними только через 1С:Предприятие или штатные средства 1С.
- **Можно:**
  - выгружать данные из 1С в XML, Excel, CSV (отчёты, обработки, обмены);
  - использовать уже настроенный HTTP-сервис (SARA/Sara-1C) для приёма заказов/ПКО из бота и мобильного приложения;
  - подключать выгрузки к проекту «Разработка медхаус» (папка **Анализ 2**, рассылка, Google Sheets и т.д.).

---

## 5. Связь с проектом управленческой отчётности (Medhouse)

- В **«Разработка медхаус»** используются:
  - **Анализ 2** — годовые файлы 2020–2025, логистика (XLSX);
  - **Рассылка из 1С** — файл «Выгрузка XLSX (2).xlsx», импорт в БД и API;
  - **Google Таблицы** — синхронизация и логирование в БД;
  - **Дашборд** — агрегаты, превью, KPI.

Данные из базы **D:\1с** попадают в этот проект **не напрямую**, а через выгрузки из 1С (например, те же XLSX рассылки и отчёты по шаблонам), которые затем загружаются или кладутся в папки/источники проекта.

Рекомендация: продолжать настраивать в 1С регламентные выгрузки (рассылка, отчёты в Excel/CSV) и при необходимости сохранять их в папку проекта или загружать через раздел «Импорт отчётов» — тогда дашборд и API будут использовать актуальные данные.

## 6. Краткая схема

```
D:\1с
├── 1Cv8.1CD, 1Cv8snc.1CD, *.cfl   ← файловая база 1С (только через 1С)
├── Medh_2026-02-24.dt             ← резерв/выгрузка DT (только через 1С)
├── ОБЗОР_ПРОЕКТА.md               ← описание SARA, бот, API 1С
├── 8.3.27.1936 / windows_*        ← платформа 1С
├── 1Cv8Log, logs                  ← журналы и логи
├── pharmacy-bot-main              ← код бота
└── Сара                           ← мобильное приложение + Sara-1C (HTTP-сервис 1С)
```

Итог: база **Medh_2026-02-24** в **D:\1с** полностью учтена по структуре и файлам; программно её «прочитать изнутри» без 1С нельзя; для сайта и отчётности используются выгрузки из 1С (XLSX, рассылка, при необходимости XML/CSV) и уже настроенные источники в проекте «Разработка медхаус».
